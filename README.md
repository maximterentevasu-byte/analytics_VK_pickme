# VK Sheets Metrics Sync v8

JS cron-сервис для Railway: собирает недельную аналитику VK, пишет в Google Sheets и готовит листы для Data Studio / Looker Studio + AI-рекомендации по контенту.

## Что пишет

- `weekly_summary` — главная недельная таблица.
- `posts_raw_vk` — сырые посты.
- `dashboard_data` — плоская таблица для подключения к Data Studio.
- `ai_recommendations` — рекомендации: что постить, когда постить, какие форматы работают лучше.

## Важно про Data Studio

API для автоматического создания готового отчёта Data Studio из скрипта не используется. v8 автоматически готовит лист `dashboard_data`, который подключается к Data Studio как источник Google Sheets. Дальше создаёшь отчёт один раз вручную и он будет обновляться по мере обновления Google Sheets.

Google в апреле 2026 вернул название Data Studio вместо Looker Studio. Старые URL/документация могут ещё называться Looker Studio.

## Railway

Start command:

```bash
npm start
```

Cron пример: каждый понедельник 09:00 UTC

```cron
0 9 * * 1
```

## Переменные

Смотри `.env.example`.

## Предупреждения

Если `stats.get` недоступен, поля охватов, посетителей, подписок и отписок останутся пустыми. Код не подставляет фейковые нули. Для ER по охвату используется fallback на просмотры постов с предупреждением.

Исторические подписчики нельзя точно восстановить, если раньше не было сохранённых недельных снимков или VK не отдаёт подписки/отписки через `stats.get`.
